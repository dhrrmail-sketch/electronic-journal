# Изменения Beta 66

Короткий отчёт по задачам из ТЗ. Сделано по приоритетам: сначала сохранность данных и честность оценок.

## Что нужно сделать вам вручную

1. Заново выполнить обновлённые SQL-файлы в своём проекте Supabase (SQL Editor → New query → Run):
   - `supabase-tests.sql`
   - `supabase-quiz.sql`
   Все функции и таблицы обновляются безопасно (`create or replace`, `add column if not exists`), данные не теряются.
2. В кабинете EmailJS включить ограничение по разрешённым доменам-источникам (Account → Security → Allowed origins). Это пара кликов на вашей стороне, из кода сделать нельзя.
3. Выполнить обновлённый `supabase-learning-materials.sql` (бакет станет приватным). После этого отправьте учебные материалы заново — старые публичные ссылки перестанут работать.
4. (По желанию) Открыть `tests.html` в браузере и убедиться, что все проверки зелёные.

## Сделано полностью

### Задача 1. Нормализация ответов на сервере (высокий приоритет)
- В `supabase-tests.sql` и `supabase-quiz.sql` добавлена функция `normalize_answer(jsonb)`: убирает дубли и пустые значения, сортирует.
- Тесты: сравнение ответа и эталона идёт через нормализацию (функция `submit_test_attempt`, разбор).
- Викторина: `quiz_answer` сравнивает нормализованные наборы.
- Итог: порядок выбора и случайные дубли больше не влияют на оценку.

### Задача 2. Частичный балл за вопросы с несколькими ответами (высокий приоритет)
- В `test_questions` добавлена колонка `scoring` (`all_or_nothing` по умолчанию, либо `partial`).
- Расчёт балла вынесен в общую функцию `score_answer(...)` — один источник правды и при сдаче, и в разборе.
- Частичный балл = (верно выбранные − лишние) / число правильных, не ниже нуля, умноженное на баллы вопроса.
- В `test-admin.html` у вопросов типа «несколько ответов» появился переключатель «Полный балл / Частичный балл».
- В `take-test.html` в разборе показывается «заработано / максимум» по каждому вопросу.
- Старые вопросы без изменения настройки оцениваются как раньше.

### Задача 3. Защита онлайн-викторины (высокий приоритет)
- В `quiz_rooms` добавлены поля `join_closes_at` (срок действия кода) и `lock_after_start` (закрыть вход после старта).
- `quiz_join`: проверяет срок кода; после старта новые участники не входят (уже вошедшие могут переподключиться).
- `quiz_roster`: после старта список группы наружу не отдаётся; при просроченном коде — отказ.
- В `quiz-admin.html` (раздел «Мои викторины») преподаватель задаёт срок кода и закрытие входа; настройки запоминаются и применяются к новым комнатам.

### Задача 6. Предупреждение о конфликте синхронизации (высокий приоритет, частично)
- Добавлено обнаружение конфликта: если после последней синхронизации данные изменились и в облаке, и на устройстве, приложение больше не перезаписывает молча.
- Появился диалог выбора: «Оставить облачную», «Оставить локальную», «Скачать копию» (резервная копия перед заменой), «Решить позже».
- При фоновой автосинхронизации конфликт не перезаписывает данные, а откладывается с уведомлением; выбор предлагается при ручной синхронизации или открытии окна «Синхронизация».
- Базовая версия отслеживается через метку `remote-seen`; идентичное содержимое конфликтом не считается.
- Полный переход на слияние по сущностям остаётся отдельным крупным этапом.

### Задача 7. Производительность (частично)
- Удалён дубль-копия `index-6873076f.html` (устаревшая Beta 65, нигде не подключалась).
- Минификация и ленивая загрузка тяжёлых разделов — следующий шаг.

## Осталось на следующие шаги (не входит в эту итерацию)

### Задача 4. Приватные учебные материалы и подписанные ссылки (высокий приоритет)
- Бакет `learning-materials` теперь **приватный** (`supabase-learning-materials.sql`): прямой публичный доступ к файлам закрыт.
- Материалы раздаются по временным подписанным ссылкам (signed URL): в письмах студентам — на 30 дней, в предпросмотре — на 6 часов, при разборе текста — на 1 час.
- Политики: преподаватель читает/загружает/удаляет только свои файлы; студенты к бакету напрямую не обращаются.
- Внешние ссылки (NotebookLM и т.п.) работают как раньше.
- ⚠️ Нужно выполнить обновлённый `supabase-learning-materials.sql` в Supabase. Ранее выданные публичные ссылки перестанут работать — отправьте материалы заново (сформируются свежие подписанные ссылки).

### Задача 8. UX и доступность
- Единый модал подтверждения (`ui-confirm.js`) вместо браузерного `confirm()` — во всех экранах (журнал, тесты, викторина, кабинет, прохождение теста). Удаление окрашено красным, фокус по умолчанию на «Отмена».
- Доступность: удержание фокуса и клавиатура (Enter/Esc/Tab) в модале; `scope="col"` и подпись у таблицы журнала; aria-подписи у полей оценок и кнопок действий.
- Визуальное выделение семинарского (секционного) столбца в журнале.
- Названия критериев по умолчанию не менял намеренно: текущие («Секционный ответ», «Индивидуальный ответ», «СРСП») осмысленны. Если хотите иные — скажите, поменяю.

### Задача 5. Безопасность контента (CSP) и аудит XSS
- Во все экраны (журнал, тесты, викторина, кабинет, прохождение теста/викторины, сдача работы) добавлен заголовок `Content-Security-Policy`.
- Разрешены только нужные источники: сам файл, jsdelivr (jszip), Google Fonts, ваш Supabase (REST/Storage/Realtime) и EmailJS. Внешние скрипты, `object/embed`, подмена `base-uri` и `eval` заблокированы.
- Аудит вставки HTML: пользовательские данные экранируются функцией `esc()` (экранирует `& < > " '`) в обоих ядрах (журнал и `test-shared.js`); опасных вставок без экранирования не найдено; `eval`/`new Function` в коде не используются.
- Примечание: из-за множества встроенных скриптов сохранён `script-src 'unsafe-inline'`; основная защита от XSS — экранирование, CSP работает как второй рубеж.

### Задача 9. Автоматические тесты
- Добавлена страница `tests.html` — откройте в браузере, тесты запускаются сами.
- 24 проверки: нормализация ответов, одиночные и «всё или ничего», частичный балл (включая штраф за лишние, дубли, округление), и решение конфликта синхронизации (pull/push/conflict/noop).
- Эталонные реализации в тестах зеркалят логику SQL (`normalize_answer`, `score_answer`) и клиентского `syncCloud` — служат и спецификацией поведения.
